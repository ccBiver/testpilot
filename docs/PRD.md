# TestPilot — AI 自主测试平台 PRD

> 版本 v0.1 · 2026-07-14 · 状态:草案

## 1. 产品定位

面向团队的自动化测试平台:AI 以真实用户视角**自主探索** Web 站点和移动 App,
自动发现缺陷并生成结构化 Bug 报告,归档到平台内看板;同时支持**导入测试用例**
逐条执行,覆盖回归测试场景。

**一句话**:给一个 URL 或 App,平台像一个不知疲倦的测试工程师一样点遍所有流程,
把 Bug 连同截图和复现步骤放到你的看板上。

## 2. 目标用户与场景

| 用户 | 场景 |
|------|------|
| QA 工程师 | 新版本发布前,提交探索任务,次日查看 Bug 看板并分派 |
| 开发者 | 提交分支部署的预览环境 URL,合并前自动冒烟 |
| 项目负责人 | 查看项目质量趋势、未关闭 Bug 数量、回归通过率 |

## 3. 产品形态与页面结构

```
公开区(未登录)                 用户控制台(登录后)              管理员后台(admin)
├── 宣传首页(产品介绍/亮点/    ├── 项目列表与项目详情            ├── 用户管理(列表/禁用/角色)
│   案例/价格占位/CTA)        ├── 发起测试(探索/回归)          ├── 全局运行监控与队列状态
├── 注册(邮箱+密码,邮箱验证   ├── 运行历史与运行详情            ├── 模型配置(可用模型/key)
│   可后置)                   ├── Bug 看板与缺陷详情            ├── 成本统计(按用户/项目)
└── 登录(含忘记密码)          └── 用例库                        └── 系统设置(注册开关等)
```

- 角色:`user`(只见自己的项目)、`admin`(后台管理 + 可见全局)。
- 宣传首页是独立营销页:产品价值、功能亮点、示例报告截图、注册入口;SEO 友好。
- **设计基调(用户确认):活泼简约、富有动效**——明快底色 + 高饱和点缀色、
  圆角卡片、大标题、入场/滚动显现/悬停微交互(framer-motion);
  按钮、输入框、加载器等控件优先从 uiverse.io(MIT)挑选动效组件适配成 React,
  避免从零手写;宣传页与控制台/后台共用同一设计语言。

## 4. 核心功能(按优先级)

### P0 — AI 自主探索(核心差异化)
- 输入:目标 URL / App 包 + 可选的探索目标描述(如「重点测试下单流程」)、
  测试账号、探索步数/时长预算。
- AI 代理基于页面截图 + 界面结构决策下一步操作:点击、输入、滑动、导航,
  像真实用户一样走完注册、登录、搜索、表单提交等流程。
- 探索过程中持续运行缺陷检测器(见 §4),每个疑似缺陷记录:截图、
  操作路径(复现步骤)、控制台/网络证据、严重级别。

### P0 — Bug 报告与平台内看板
- 每次运行生成一份结构化报告:缺陷列表、通过的流程、覆盖的页面地图。
- Bug 看板:按项目归档,支持严重级别(Critical/High/Medium/Low)、
  状态流转(待确认 → 已确认 → 修复中 → 已关闭 / 误报)、去重合并。
- 缺陷详情页:截图、逐步复现路径、控制台错误、失败的网络请求、环境信息。

### P1 — 测试用例导入与回归执行
- 支持 YAML / Excel / 自然语言用例(「打开首页,搜索 iPhone,断言结果非空」)。
- AI 逐条解释执行,输出 通过/失败/阻塞 + 失败证据。
- 用例库按项目管理,可标记为回归集,发版前一键全量回归。

### P0 — 账户体系、宣传页与后台管理
- 宣传首页(公开):产品介绍 + 注册转化入口。
- 注册/登录:邮箱 + 密码,JWT 会话,忘记密码;注册开关由后台控制。
- 数据隔离:用户只能看到自己的项目与运行。
- 管理员后台:用户管理、全局运行监控、模型与 key 配置、成本统计。

### P1 — 任务调度与历史
- 手动触发、定时触发(cron)、Webhook 触发(CI 部署完成后自动测)。
- 运行历史、质量趋势图(缺陷数、回归通过率随时间变化)。

### P2 — 后续
- 团队/组织(项目共享给成员、成员角色细分)。
- 第三方推送:GitHub Issue / Jira / 飞书通知。
- 录制回放生成用例;iOS 支持完善。

## 5. 缺陷检测维度(探索模式)

| 检测器 | 信号 | 默认级别 |
|--------|------|----------|
| 崩溃/白屏 | 页面 crash、Android ANR/FC(logcat)、渲染空白 | Critical |
| JS/运行时错误 | console.error、未捕获异常 | High |
| 网络故障 | 4xx/5xx、超时、接口报错体 | High |
| 流程阻塞 | 按钮点击无响应、表单无法提交、死链 | High |
| AI 视觉判定 | 布局错乱、文案溢出、加载态卡死、明显 UI 异常 | Medium |
| 体验问题 | 无反馈的慢操作、错误提示不友好 | Low |

## 6. 范围与非目标(v1)

- ✅ Web(桌面 + 移动视口)、Android(模拟器/真机 via adb)。
- ✅ 中文优先的界面与报告。
- ❌ v1 不做:iOS(M4)、性能压测、安全渗透测试(与 AppShield 边界区分)、
  单元/接口测试(只做 UI 层)。

## 7. 成功指标

- MVP:对 3 个真实项目运行探索,能发现人工已知 Bug 的 ≥60%,误报率 <30%。
- 单次 50 步探索成本可控(模型调用费用有预算上限并在报告中展示)。
- 用例回归:导入 20 条自然语言用例,执行判定准确率 ≥90%。

## 8. 关键风险

| 风险 | 缓解 |
|------|------|
| AI 误报多、报告不可信 | 缺陷需附客观证据(console/网络/截图对比);AI 判定类默认 Medium 以下并标注置信度 |
| 探索破坏性操作(删数据、真实下单) | 强制测试环境声明;敏感操作词黑名单;只读模式开关 |
| 模型成本失控 | 每次运行设 token/步数预算,超限即止并出报告 |
| 双端并行导致战线过长 | 执行层统一走 Midscene 抽象,M0 只验证 Web,Android 在 M2 接入 |
