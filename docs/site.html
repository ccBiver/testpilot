Pre-flight · 怎么用

登机三步

装一次,之后任意目录敲 testpilot,跟着向导选就行——不用记参数。

STEP 01

装一次

在仓库里跑 pnpm install 和 pnpm install:cli,把 testpilot 装成全局命令。

STEP 02

进你的项目

cd 到你要测的项目目录(报告和用例会落在这里),敲 testpilot。

STEP 03

跟着向导选

做什么 → 平台 → 目标 → 深度。回车,AI 开测,完事出 HTML 报告。

testpilot — 交互向导
Status Board · 平台支持

三端真实状态

不夸大:每一端要准备什么、能不能零成本,列清楚。

平台探索 / 用例零成本(本机 Claude)前置准备
Web 全功能 ✓ 是 无(浏览器自动装)
Android 全功能 ✓ 是 开模拟器 / 连真机 + adb
iOS 探索 / 用例 需模型 key macOS + Xcode + WDA@:8100

零成本怎么来的: Web / Android 用你本机的 Claude Code 订阅做决策(看截图选元素),不走付费 API。 iOS 因未暴露公开坐标点击,走视觉模型(如 DashScope Qwen-VL),需要一个模型 key。

Manual Control · 命令参考

老手直接敲(可脚本化 / CI)

testpilot交互向导 · 新手推荐
testpilot explore <url>AI 探索 Web 站点
testpilot explore-app <包名>探索 Android 应用
testpilot explore-ios <bundleId>探索 iOS 应用 · macOS
testpilot gen-cases prd.md -t <url>需求文档 → 用例
testpilot gen-cases --figma <链接> -t <url>Figma → 用例
testpilot run-cases cases.yaml执行用例文件

提示: 向导里生成完用例会直接问「马上执行?」,gen → run 一步串起,平时根本不用记这些。
